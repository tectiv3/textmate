# LSP Phase 1: Foundation + Diagnostics — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extract the PoC LSP client into `Frameworks/lsp/`, add `.tm_properties` config, document sync, and diagnostics in the gutter.

**Architecture:** LSPClient (JSON-RPC + process), LSPManager (singleton, config → client mapping, document lifecycle), and diagnostics stored as buffer marks using existing `error`/`warning`/`note` mark types. OakDocumentView wires notifications to LSPManager.

**Tech Stack:** ObjC++, nlohmann/json (already vendored), NSTask/NSPipe, libdispatch, existing buffer marks system

**Spec:** `docs/superpowers/specs/2026-03-14-lsp-phase1-design.md`

### Task 3: Implement LSPManager

**Files:**
- Create: `Frameworks/lsp/src/LSPManager.h`
- Create: `Frameworks/lsp/src/LSPManager.mm`

LSPManager is the central coordinator. Refer to the spec for full details. Key implementation notes:

- [ ] **Step 1:** Create `LSPManager.h` — singleton with `sharedManager`, methods `documentDidOpen:`, `documentDidChange:`, `documentDidSave:`, `documentWillClose:`, `shutdownAll`.

- [ ] **Step 2:** Create `LSPManager.mm`. Internal state:
  - `_clients`: root path → LSPClient
  - `_documentClients`: document UUID → LSPClient (reverse lookup)
  - `_documentVersions`: document UUID → int version counter
  - `_openDocuments`: set of document UUIDs (prevents duplicate didOpen from split views)
  - `_changeTimers`: document UUID → NSTimer (300ms debounce for didChange)

- [ ] **Step 3:** Implement `clientForDocument:` — reads `lspCommand` and `lspEnabled` from `settings_for_path()`, determines workspace root (spec has the marker list and `lspRootPath` override), parses command string (split on whitespace), creates or reuses LSPClient per root.

- [ ] **Step 4:** Implement workspace root detection as a free function. Walk up from file directory checking for markers: `.git`, `composer.json`, `package.json`, `tsconfig.json`, `CMakeLists.txt`, `compile_commands.json`, `go.mod`, `Cargo.toml`, `pyproject.toml`, `setup.py`, `.clangd`. Use `path::exists()` and `path::parent()` from `<io/io.h>`.

- [ ] **Step 5:** Implement scope → languageId mapping. Static NSDictionary, fallback: strip `source.` prefix. See spec for full mapping table.

- [ ] **Step 6:** Implement document lifecycle methods with debounced didChange (300ms NSTimer). `documentDidSave:` should flush pending didChange first. `documentWillClose:` should clean up all tracking state.

- [ ] **Step 7:** Implement `LSPClientDelegate` — on `didReceiveDiagnostics:forDocumentURI:`, convert URI to path, find document via `[OakDocument documentWithPath:]`, call `removeAllMarksOfType:` for `error`, `warning`, `note`, then `setMarkOfType:atPosition:content:` for each diagnostic. Map LSP severity: 1→`error`, 2→`warning`, 3/4→`note`. Position: `text::pos_t(line, 0)`.

- [ ] **Step 8:** Build and verify. Commit.
